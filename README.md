# Predictive Maintenance System for CNC Machines

ML-based system for early detection of equipment failures using sensor telemetry. Predicts failures 8+ hours in advance, reducing unplanned downtime costs.

## Architecture

```
Raw Sensor Data -> Preprocessing -> Feature Engineering -> LightGBM Classifier -> Alerts
                                      ├── Statistical features (rolling windows)
                                      ├── Spectral features (FFT, PSD)
                                      └── TimesNet embeddings (optional)
```

Key design decisions:
- Time-series cross-validation to prevent look-ahead bias
- Cost-sensitive threshold tuning ($500 FP, $10,000 FN)
- Ring buffer for real-time inference (< 5ms latency)
- MLflow tracking for experiment reproducibility

## Dataset

Uses a synthetic dataset mimicking real CNC machine telemetry generated by `scripts/download_data.py`:

| Parameter | Value |
|-----------|-------|
| Machines | 10 |
| Sensors | volt, rotate, pressure, vibration |
| Sampling rate | 5 minutes |
| Time range | 1 year |
| Failure types | 5 components |
| Class imbalance | 1:1000 (healthy vs failure) |

The generator introduces:
- Gaussian sensor noise
- Random walk drift (sensor aging simulation)
- Exponential vibration growth before failures

## Usage

### Training pipeline

```bash
# Basic training
python scripts/train_pipeline.py

# With hyperparameter tuning
python scripts/train_pipeline.py --cross-validate

# Without TimesNet embeddings (faster)
python scripts/train_pipeline.py --skip-timesnet
```

Training steps:
1. Load raw telemetry and error logs
2. Clean outliers, interpolate missing values
3. Label failure windows (merge errors with telemetry)
4. Augment data (noise, drift, imbalance)
5. Extract features (statistical, spectral, deep embeddings)
6. Time-based train/val/test split
7. Train LightGBM with cost-sensitive threshold
8. Save model and log metrics to MLflow

### Real-time inference

```python
from src.deployment.inference import InferencePipeline

pipeline = InferencePipeline(
    model_path="models/lgbm_model.joblib",
    alert_threshold=0.3
)

measurements = {
    "volt": 170.0,
    "rotate": 450.0,
    "pressure": 100.0,
    "vibration": 40.0

```

## Feature Engineering

### Statistical Features (rolling windows: 5min, 30min, 2h)
- Mean, standard deviation, min, max
- Kurtosis, skewness
- 25th and 75th percentiles

### Spectral Features (vibration only)
- Top-10 dominant frequencies via FFT
- Power spectral density (Welch method)

### TimesNet Embeddings (optional)
Pretrained 1D-CNN encoder that transforms raw sensor windows into 512-dimensional embeddings. Captures complex non-linear patterns that handcrafted features may miss.

## Evaluation Metrics

| Metric | Target |
|--------|--------|
| Recall (failure detection) | >= 0.93 |
| False positive rate | < 0.05 |
| Alert horizon | 8-24 hours |
| Inference latency (p95) | < 5ms |

### Cost-Sensitive Metric

The model optimizes for business value, not accuracy:

| Outcome | Cost/Gain |
|---------|-----------|
| True Positive (caught failure) | +$8,000 |
| False Positive (unnecessary dispatch) | -$500 |
| False Negative (missed failure) | -$10,000 |

The prediction threshold is tuned to maximize total cost savings on a validation set.

## Model Card

- **Algorithm**: LightGBM (gradient boosting)
- **Objective**: multiclass classification (5 failure types + healthy)
- **Class balancing**: weighted classes + undersampling
- **Validation**: Blocked time-series cross-validation (5 folds)
- **Hyperparameter tuning**: Optuna with MedianPruner
- **Feature importance**: Gain-based ranking

## Limitations

- TimesNet embeddings require pretrained weights (placeholder CNN used if file not found)
- Synthetic data may not capture all real-world failure patterns
- Model requires at least 5 minutes of sensor history for first prediction
- Cold start problem: new machines need transfer learning approach
- Threshold should be recalibrated if cost structure changes
